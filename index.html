ONTARIO PARKS

government website redesign

Project Overview

Ontario provincial parks website, the largest provider of outdoor recreation experiences in Ontario, Canada. The website provides reservation system for booking individual campsites, group camping areas or comfort camping and equipments., it offers education programs about nature and heritage. also it is the part of local communities promoting Ontario tourism.

Current website has not been updated for a while, the new reservation system was launched 6 months ago, but still has some shortcomings . There are overwhelming informations on the site, no search function existed. For the first time users, it is hard to know where to start to find a park with outdoor activities they search for. And the original site is not responsive web design.


The Solution

Building a meaningful user interface from a clear and deliberate brand voice for a government agency website, it only provide essential information and easy for users to navigate and find the relevant information they search.


Deliverables

UI style guide, information architecture /site map, sketches &wireframes, and responsive high-fidelity mock-ups.

My Role

  • UX/UI designer
  • Skills Applied: Research,user interviews, wireframing, prototyping, usability tests, and visual design
  • Timeline: 4 weeks

My Process:

  • Research
  • Rebrand
  • Prototyping & test
  • RWD wireframes
  • User test & iterations

RESEARCH

Research Method

1. Identify the user (proto persona) and user scenario
2. Feature Prioritization

After defined the user and user scenario, I conducted 5 guerrilla usability tests of current site, and mapping & prioritizing key insights found through 5 usabilities testing.

3. Redline & Annotation

I performed website anlaysis and annotated usability & heuristic issues for current website.

REBRAND

Rebranding Approach

Starting with organizing a project’s inspiration -mood board, creating a new site map after card sorting activity, and stylizing UI elements to define new structural & visual rules.

1. Mood Board

I found that creating a mood board is extremely useful for establishing the aesthetic feel of a website.

2. Sitemap

I referenced results from the card sorting activity to identify content patterns, group like items together, and label new categories. By removing duplicate navigation items and creating tertiary navigation, I was able to build a sitemap that more structured and comprehensive.

3. UI Style Tile

One of my main redesign solutions is to rebrand and have a new face for the site. I created their new logo, chose the fonts, colors and pictures for the new site - stylizing UI Elements.

PROTOTYPE & TEST

1. Lo-Fi Wireframe

The goal of the low-fidelity wireframe was to determine the content structure on each page before applying the visual design. Based on site audit results and redesigned site maps, I reorganized Nav categories and add new category as needed, rearanged footer items, redesigned homepage layout by adding new sections and search function, redesigned reservation page and FAQ page, and also added two new pages -”find a park” and “contact us” page.

2. 5-Second User Test

Using the low-fidelity prototypes, I conducted 5-second user testing with 3 participants to determine if users’ first impressions of my redesigned pages are on point, and if both top and footer nav work. Users feedback were positive: “Both top and footer Nav are clear and easy to navigate. Nice page layout. Clear on button states. Could consider to change search field ( no need to click twice to get to search bar, etc.)".

I iterated lo-fi wireframes based on all the comments received from testing before I turned it to high fidelity.

RWD WIREFRAMES

1. UI Style Guide

Before I turned my design into high fidelity, I expanded my UI style tile to UI style guide - new colors, new fonts, new designed logo and consistent UI elements to apply to all the wireframes.

2. Responsive Web Design

I applied UI style guide to all the wireframes to bring the design to life, and created the responsive web design to fit different screen sizes, turned them into the hi-fidelity prototypes.

USER TEST & ITERATIONS

User Testing Plan & Notes

Using the high-fidelity prototype, I conducted the usability tests with 7 participants for the mobile site. The main goals were to find out if mobile site is organized the way easily for user to navigate and find their information.

FINAL PTOTOTYPE

Final Desktop & Mobile Prototype

After multiple iterations based on feedbacks received from user test, here are final RWD wireframes.I have addressed all issues noted in the original site audit and usability test - rebrand the organization (new logo, new font, colors and pictures for the site) , redesigned home page, reservation page & FAQ page, added “Find a Park” page and “Contact Us “page, rearranged Nav items and footer items, added search funtion to the site, at last I created the responsive web design.
